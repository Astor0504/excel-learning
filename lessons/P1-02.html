3 個最容易搞混的地方

COUNT 跟 COUNTA 差在哪？

COUNT 只會理會「數字」；COUNTA 則是只要不是空白就算進去，包含文字、日期、0、公式回傳的內容。你如果在算「有幾筆業績金額」，通常用 COUNT；如果在算「有幾位員工有填資料」，通常用 COUNTA。

AVERAGE 怎麼看 0、空白、文字？

AVERAGE / COUNT / COUNTA 的處理差異
儲存格內容COUNTCOUNTAAVERAGE
120000算算算
0算算算
空白不算不算不算
文字「待補」不算算不算

所以如果資料是 120000、0、空白、待補、95000，AVERAGE 會用 (120000 + 0 + 95000) ÷ 3。0 不是空白，它會真的把平均拉低。

MAX / MIN 跟 LARGE / SMALL 怎麼分工？

MAX / MIN 只回答「最大值」和「最小值」；LARGE / SMALL 回答的是「第幾名」。問「最高業績多少」用 MAX，問「第 3 名業績多少」才用 LARGE。倒數第 2 名、倒數第 3 名，就換 SMALL。

實戰範例：同一張業績表能練熟的基本動作

# 業績統計（A欄=業務員，B欄=金額）
=SUM(B:B)              # 全部業績
=AVERAGE(B2:B100)      # 平均單筆
=COUNT(B:B)            # 有幾筆業績
=COUNTA(A:A) - 1       # 業務員人數（扣標題）
=MAX(B:B)              # 最高一筆
=LARGE(B:B, 3)         # 第 3 高（找前三名）
⚠️ 本課真正要記住先問自己：你要的是總和、平均、筆數、還是名次？這個問題問對了，函數就不容易選錯。平均值怪怪的時，第一件事先檢查資料裡是不是有 0、空白或文字。
