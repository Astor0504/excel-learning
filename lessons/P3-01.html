資料驗證

本課進度 0 / 4
TL;DR — 30 秒看懂

資料驗證：讓使用者只能輸入合法的值

資料驗證的價值，是把錯誤擋在輸入當下。與其月底再清「2024/13/45」「電子部電子部電子部」這種髒資料，不如一開始就讓使用者只能填合法值。

先決定要防什麼錯

資料驗證類型速查
要防的錯驗證類型例子
選項打錯或同義字混用清單部門只能選電子/業務/行政
數字超出合理範圍整數 / 小數折扣介於 0% 到 30%
日期不在期間內日期只能填本季日期
格式或跨欄條件自訂公式結束日必須晚於開始日

最常用：下拉選單

資料 → 資料驗證 → 允許「清單」→ 來源輸入 電子,業務,行政（用半形逗號），或選一個維護清單的範圍。正式表單建議用範圍來源，因為之後新增選項時不必逐格修改規則。

日期、數字與錯誤提醒

數字與日期驗證要同時設定「輸入訊息」和「錯誤提醒」。輸入訊息告訴使用者該填什麼，錯誤提醒則說明為什麼被擋下。這兩個訊息寫清楚，表單才不會變成只會跳錯的黑盒子。

自訂公式：跨欄防呆

# 結束日必須晚於開始日
=B2>=A2

# 員工編號不可重複
=COUNTIF($A:$A, A2)=1

# 金額必須大於 0，且狀態不可空白
=AND($C2>0, $D2<>"")

串聯下拉（進階）

# 第一個下拉選「部門」
# 第二個下拉根據第一個顯示對應「員工」
=INDIRECT(A2)
# A2 是「電子」就會顯示名為「電子」的範圍

正式表單的 5 步流程

不要只做下拉選單，整份表單要一起設計
步驟目的交付檢查
1. 先列輸入欄位分出必填、選填、公式產生欄公式欄不應讓使用者手填
2. 建維護清單部門、區域、狀態集中管理新增選項時不用逐格改規則
3. 套驗證規則清單、日期、數字、自訂公式分工每種規則都測合法與非法輸入
4. 寫提示與錯誤訊息讓使用者知道該怎麼填錯誤訊息要說明原因，而不是只說不允許
5. 圈選舊資料找出套規則前已經存在的髒資料資料 → 資料驗證 → 圈選無效資料

什麼時候資料驗證不夠

資料驗證只能擋住儲存格輸入錯誤，不能取代權限控管，也不能保證貼上資料一定乾淨。若檔案會長期給多人使用，請搭配 Table、條件式格式化和工作表保護：Table 讓範圍穩，條件式格式化把異常浮出來，保護工作表避免使用者改到公式。

⚠️ 注意資料驗證只管「之後輸入」的值，不會自動修正既有髒資料。套用前先清一次資料，套用後再用「圈選無效資料」檢查漏網內容。
