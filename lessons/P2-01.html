條件統計

本課進度 0 / 4
TL;DR — 30 秒看懂

條件統計先問兩件事：要算筆數、加總、還是平均？條件是一個還是多個？答案會直接決定 COUNTIF/SUMIF/AVERAGEIF 或 COUNTIFS/SUMIFS/AVERAGEIFS。

條件統計不是六個函數分開背，而是一條固定判斷路徑：先看要算筆數、合計還是平均；再看條件是一個還是多個；最後才寫語法與驗算命中列。

本課解法路徑

遇到條件統計題，照這 4 步走
步驟你要回答結果
1. 算什麼幾筆、合計、還是平均？COUNT / SUM / AVERAGE 家族
2. 條件幾個一個條件，還是多個條件同時成立？IF 或 IFS 版本
3. 範圍順序先寫條件範圍，還是先寫計算範圍？避免 SUMIF / SUMIFS 順序寫反
4. 驗算命中列哪些列真的被算進去？確認結果可以交付

第一步：先選公式家族

主管問法單一條件多個條件你在算什麼
北區有幾筆訂單？COUNTIFCOUNTIFS符合條件的筆數
電子類金額合計？SUMIFSUMIFS符合條件的數字合計
北區電子平均金額？AVERAGEIFAVERAGEIFS符合條件的數字平均

這張表是本課的核心。先不要急著背括號裡的參數，先把問題翻成「算筆數 / 算合計 / 算平均」和「單條件 / 多條件」。選對公式後，語法只剩下照順序填範圍。

第二步：單條件先抓條件欄

# 數筆數：北區有幾筆訂單？
=COUNTIF(C:C, "北區")

# 加總：電子類別的金額合計？
=SUMIF(D:D, "電子", E:E)

# 平均：業務部平均績效？
=AVERAGEIF(B:B, "業務", D:D)

單條件的思路是先指定「去哪一欄檢查條件」。COUNTIF 數命中的格數；SUMIF / AVERAGEIF 會再去同列的數字欄做加總或平均。

第三步：多條件是一組一組篩

# 多條件計數：北區且金額 >= 15000 的訂單數
=COUNTIFS(C:C, "北區", E:E, ">=15000")

# 多條件加總：電子且已付款的金額合計
=SUMIFS(E:E, D:D, "電子", F:F, "已付")

# 多條件平均：北區電子類的平均金額
=AVERAGEIFS(E:E, C:C, "北區", D:D, "電子")

所有 IFS 版本都是 AND 關係：每個條件都要同時成立。每一組條件都要寫成「條件範圍、條件值」，而且所有範圍列數必須一致。

第四步：專門記住範圍順序

# SUMIF：條件範圍在前，加總範圍在後
=SUMIF(條件範圍, 條件, 加總範圍)

# SUMIFS：加總範圍放第一個，後面才是一組一組條件
=SUMIFS(加總範圍, 條件範圍1, 條件1, 條件範圍2, 條件2, ...)

# AVERAGEIFS 也一樣：平均範圍放第一個
=AVERAGEIFS(平均範圍, 條件範圍1, 條件1, 條件範圍2, 條件2, ...)
注意本課最常見錯誤就是把 SUMIF 和 SUMIFS 的順序混在一起。單條件 SUMIF 是「先條件、後加總」；多條件 SUMIFS 是「先加總、後條件」。

條件寫法：不只等於

=COUNTIF(B:B, "王*")          # 開頭是「王」
=COUNTIF(D:D, "*電子*")        # 包含「電子」
=COUNTIF(F:F, "<>已付")       # 不等於已付
=COUNTIF(E:E, ">=15000")       # 大於等於 15000
=SUMIFS(E:E, C:C, "北區", E:E, ">=15000")

* 代表任意長度文字，? 代表單一字元。比較運算子如 >=、<> 通常要放在引號裡；如果門檻放在 H2，寫成 ">="&H2。

交付報表前檢查

  • 先手動篩選一次資料，確認公式命中的列和你預期一致。
  • 條件欄和計算欄的列數是否一致；不要一欄抓到 100 列、另一欄抓到 99 列。
  • 文字條件有沒有多餘空白、全形半形或狀態名稱不一致。
  • 日期條件是否是真正日期，而不是看起來像日期的文字。
  • 多條件預設是 AND；如果需求是 OR，通常要把多個 COUNTIFS 或 SUMIFS 相加。
WPS Office 條件統計函數相容性
  • SUMIF、SUMIFS、COUNTIF、COUNTIFS、AVERAGEIF、AVERAGEIFS 語法與 Excel 完全相同
  • 萬用字元（*、?）行為一致，可直接沿用
  • 日期條件格式建議用 DATE() 而非文字，以確保跨平台一致
