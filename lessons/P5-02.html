VBA 進階：陣列、字典、錯誤處理、事件

進階 VBA 的重點不是寫更花的程式，而是讓巨集更快、更穩、更容易收尾。先把大量資料放進記憶體處理，再用 Dictionary 做彙總，最後用錯誤處理確保設定會復原。

先判斷：為什麼需要進階 VBA

從會跑，到可以交付
問題進階技巧交付價值
逐格處理很慢陣列一次讀寫大量資料從幾分鐘降到幾秒
要去重、計數、彙總Dictionary比反覆 COUNTIF 更有彈性
程式中途壞掉On Error + CleanUp能提示錯誤並復原畫面、事件、計算設定
使用者輸入後要自動補資料Worksheet 事件不用要求使用者再按一次巨集

陣列加速

' 慢：直接讀寫儲存格（每次都跨應用程式）
For i = 2 To 10000
    Cells(i, 2).Value = Cells(i, 1).Value * 2
Next i

' 快：先讀進陣列，運算完一次寫回（快 100 倍）
Dim arr As Variant
arr = Range("A2:A10000").Value
For i = 1 To UBound(arr)
    arr(i, 1) = arr(i, 1) * 2
Next i
Range("B2:B10000").Value = arr

Dictionary（去重 / 計數神器）

Dim dict As Object
Set dict = CreateObject("Scripting.Dictionary")

For i = 2 To 1000
    Dim key As String
    key = Cells(i, 1).Value
    If dict.Exists(key) Then
        dict(key) = dict(key) + 1
    Else
        dict.Add key, 1
    End If
Next i

錯誤處理與安全收尾

Sub Safe()
    On Error GoTo ErrHandler

    Application.ScreenUpdating = False
    Application.EnableEvents = False

    ' 你的主要流程

CleanUp:
    Application.EnableEvents = True
    Application.ScreenUpdating = True
    If Err.Number = 0 Then MsgBox "完成"
    Exit Sub

ErrHandler:
    MsgBox "錯誤：" & Err.Description
    Resume CleanUp
End Sub

事件驅動

工作表可以掛事件：Worksheet_Change（資料變更時觸發）、Worksheet_SelectionChange（選取變更時）。可以做「自動填入時間戳記」之類的功能。事件程式碼要放在工作表模組，不是一般 Module；如果事件裡會改儲存格，請先關閉 Application.EnableEvents，避免觸發自己造成無限迴圈。

macOS 上的進階 VBA 邊界

Excel for Mac 可以做工作簿內資料處理、格式化、匯出 PDF、刷新查詢和基本檔案流程；但不要把 Windows 專屬的 COM / ActiveX / Outlook 自動化當成主路線。若需求是清理資料，先問 Power Query 能不能處理；若需求是按鈕化交付流程，再把 VBA 接進來。

交付前檢查

進階 VBA 不是只測成功路徑
檢查項目原因
大量資料是否一次讀寫避免逐格操作造成卡頓
錯誤時設定是否復原ScreenUpdating、EnableEvents、Calculation 不該永久被關掉
事件是否避免遞迴觸發Worksheet_Change 修改儲存格時最容易出問題
Mac / Windows 差異是否標清楚避免交付後使用者照著不可用的功能操作
WPS 使用者注意WPS 免費版不含 VBA 模組，這課的 VBA 範例在 WPS 裡跑不動——但不用裝任何元件：WPS 內建的「JS 宏」就是官方的自動化路線，物件模型跟 VBA 同一套，寫法平移方式見番外課 P5-05。想照本課原樣練 VBA，請用 Microsoft Excel。
