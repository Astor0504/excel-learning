實戰專案不是把巨集寫越長越好，而是把「輸入、處理、輸出、驗證」四段責任切清楚。這樣流程壞掉時才知道該查哪裡，也比較能交給別人使用。

自動化流程總覽

範例：月報交付自動化

工具分工提醒這個範例的重點是 VBA 負責的部分：觸發更新 → 控制輸出 → 留下交付紀錄。清資料（刪空白、重複值、格式整形）應該先在 Power Query 裡建好可重跑步驟，再讓 VBA 呼叫 RefreshAll。如果你把清資料邏輯寫進 VBA，每次流程變動就要改巨集，不如讓 Power Query 負責「清」，VBA 負責「跑」。
Sub 月報自動化()
    On Error GoTo ErrHandler

    Dim exportPath As String
    exportPath = ThisWorkbook.Path & Application.PathSeparator & _
        "月報_" & Format(Date, "yyyymm") & ".pdf"

    Application.ScreenUpdating = False
    Application.EnableEvents = False

    ' 1. 重新整理所有 Power Query 查詢（清資料邏輯留在 PQ）
    ThisWorkbook.RefreshAll

    ' 2. 等待重整完成（若有非同步查詢）
    Application.CalculateUntilAsyncQueriesDone

    ' 3. 切到月報工作表，輸出 PDF
    ThisWorkbook.Worksheets("月報").ExportAsFixedFormat _
        Type:=xlTypePDF, _
        Filename:=exportPath

CleanUp:
    Application.EnableEvents = True
    Application.ScreenUpdating = True
    If Err.Number = 0 Then MsgBox "月報已輸出：" & vbCrLf & exportPath
    Exit Sub

ErrHandler:
    MsgBox "月報流程失敗：" & Err.Description, vbCritical
    Resume CleanUp
End Sub

這個結構讓 VBA 只負責觸發與交付，清資料的責任留在 Power Query。錯誤處理也很重要：即使流程失敗，也要把 ScreenUpdating / EnableEvents 復原，避免使用者的 Excel 卡在奇怪狀態。

交付前檢查表

檢查項目為什麼重要
檔名規則固定避免每月人工改檔名造成漏報
輸出路徑可預期使用者知道 PDF 會去哪裡
錯誤訊息可讀流程失敗時能知道是缺表、缺檔還是查詢失敗
設定會復原ScreenUpdating、EnableEvents、Calculation 不該永久被關掉

加速三件事

① Application.ScreenUpdating = False 關閉畫面更新。② 大量公式重算時才暫停自動計算，並在結束時復原。③ 大量逐格處理時，先把範圍讀進陣列再一次寫回。

按鈕綁巨集

開發人員 → 插入 → 按鈕 → 拖出來 → 指定巨集。按鈕文字請寫使用者看得懂的動作，例如「更新並輸出月報」，不要只寫「Run」。

macOS 提醒Mac Excel 可以跑這種 RefreshAll、PDF 匯出、工作表按鈕流程；但不要把 Windows Outlook COM、自訂 Ribbon 或 UserForm 當成主線。這門課採用 Mac 也能交付的路線。
WPS 使用者注意承前課：WPS 免費版要先裝好「VBA 支援元件」才能跑巨集。動手做這個交付專案前，先確認環境就緒（或改用 Microsoft Excel），免得程式寫好了卻按不動。
