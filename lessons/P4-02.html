進階函式

本課進度 0 / 4
TL;DR — 30 秒看懂

進階函式不是炫技，而是把長公式變得可讀、可測、可重用

這一課不是把冷門函數塞滿腦袋，而是學會判斷：公式太長時怎麼拆？同一段邏輯重複出現時怎麼封裝？交付給別人時哪些函數會有風險？

先把工具分成 4 類

進階函式的使用場景
需求優先工具一句提醒
公式太長、重複計算太多LET替中間結果命名，讓公式像分段說明
同一段商業邏輯會重複用LAMBDA先測公式，再放進名稱管理員變成自訂函式
統計時要忽略錯誤或隱藏列AGGREGATE比 SUM / AVERAGE 多了錯誤處理選項
依代碼回傳不同結果SWITCH比多層 IF 更像對照表，維護成本較低
用文字組參照或動態範圍INDIRECT / OFFSET可用，但屬於 volatile 函數，檔案大時要節制

語法起手式

# LET：替中間結果命名，最後一段是回傳值
=LET(total, SUM(B2:E2), bonus, IF(total>=500000, total*5%, 0), total+bonus)

# LAMBDA：先用立即執行測試，再放到名稱管理員
=LAMBDA(amount, rate, amount*(1+rate))(120000, 5%)

# AGGREGATE：忽略錯誤值求平均
=AGGREGATE(1, 6, B2:B100)

# SWITCH：用代碼對應結果，少寫多層 IF
=SWITCH(A2, "A", "優先", "B", "一般", "其他")

# INDIRECT / OFFSET：能做動態參照，但大型檔案要小心效能
=INDIRECT("業績表!" & A2)
=OFFSET(A1, 0, 0, COUNTA(A:A), 1)

何時該重構成 LET？

當同一段計算在公式裡出現兩次以上，或公式已經長到你要左右捲動才看得完，就該考慮 LET。LET 的好處不是「比較高級」，而是讓下一個維護的人能看懂 total、bonus、result 各自代表什麼。

LAMBDA 的學習順序

先在儲存格裡用一般公式測通，再改成 LAMBDA 立即執行版本，最後才放進名稱管理員。不要一開始就把還沒測過的公式封裝起來，否則除錯會變得很痛苦。

兩個高風險函數：INDIRECT / OFFSET

這兩個函數能做動態參照，但它們是 volatile 函數，工作表重算時容易被反覆觸發。小表可以用；大型報表或多人協作檔，請優先考慮 Table 結構化參照、XLOOKUP、FILTER 或 Power Query。

實戰範例：季度業績公式重構

# 目標：算全年業績，達標 500,000 時加 5% 獎金
=LET(
  total, SUM(B2:E2),
  bonus, IF(total>=500000, total*5%, 0),
  total+bonus
)

# 若這段邏輯很多地方會用，再封裝成 LAMBDA
=LAMBDA(q1, q2, q3, q4,
  LET(total, SUM(q1, q2, q3, q4), bonus, IF(total>=500000, total*5%, 0), total+bonus)
)
⚠️ 交付提醒進階函數要先確認版本。LET / LAMBDA 需要較新的 Microsoft 365 / Excel 2021+ 環境；若收件者用舊版 Excel，請先準備替代方案或把結果值貼上。
WPS 使用者注意承 P4-01 的提醒：LAMBDA 在 WPS 尚未支援，本課的 LAMBDA 與 BYROW / REDUCE 範例在 WPS 會回 #NAME? 錯誤。用 WPS 的話這課先當概念課讀，實作請用 Microsoft 365 的 Excel 跟著做。
