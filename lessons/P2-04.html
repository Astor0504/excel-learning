條件式格式化：讓重要的資料自己跳出來

條件式格式化不是把報表塗漂亮，而是把「需要採取行動」的資料自動推到眼前。好的規則會回答一個明確問題：誰超標？誰逾期？哪個數字正在變差？

先判斷目的，再選規則

條件式格式化決策表
判斷目的建議規則適合場景
比較大小資料橫條業績、庫存、完成率
看高低分布色階熱力圖、風險分數、毛利率
標出狀態圖示集上升/下降、紅黃綠燈
依多欄判斷整列使用公式決定格式逾期訂單、需追蹤客戶、異常交易

完整流程：範圍、規則、格式

先選「要被上色的範圍」，再建立規則。這個順序很重要：如果你只選 D 欄，公式即使判斷正確，也只會把 D 欄上色；若要整列提醒，就要先選整個資料區，例如 A2:F100。

自訂公式規則（最強）

# 整列上色：當 D 欄是「逾期」就把 A:F 整列變紅
# 先選 A2:F100 → 條件式格式化 → 新增規則 → 使用公式
=$D2="逾期"
# $D2 = 鎖欄不鎖列：每一列都看自己的 D 欄

# 高亮週末
=WEEKDAY($A2, 2)>=6

# 標記高於平均
=B2>AVERAGE($B$2:$B$100)

# 同時符合：客訴 >= 3 且成長率 < 0
=AND($E2>=3, $F2<0)

管理規則順序

規則多了之後，請進「管理規則」檢查三件事：套用範圍是否正確、規則順序是否合理、是否需要勾選「若為 True 則停止」。同一格同時被多個規則命中時，最後看到的格式可能不是你以為的那個。

一個穩定的規則設計流程

從商業訊號到可交付格式
步驟你要先決定檢查方式
1. 定義訊號這個顏色代表要追蹤、表揚、還是修正？用一句話說清楚，不清楚就不要上色
2. 選套用範圍只標單格，還是整列都要提醒？管理規則裡檢查「套用到」是否涵蓋整個資料區
3. 寫規則單欄大小比較、文字狀態，還是多欄 AND 條件？用 2~3 列測試資料確認命中與排除都正確
4. 控制視覺層級哪個規則最重要？是否需要停止為 True？避免每格同時有 3 種顏色，最後反而看不出重點

交付前檢查

條件式格式化做好後，請刻意新增一筆「應該命中」和一筆「不該命中」的資料，確認格式會跟著反應。正式報表要把規則名稱或判斷邏輯寫在旁邊的小註記，否則下個接手的人只會看到一堆顏色，不知道它們代表什麼。

⚠️ 注意公式裡 $ 用錯地方是最常犯的錯。整列上色通常要鎖欄不鎖列，例如 $D2；如果寫成 $D$2，整張表都只會看 D2，一改就全錯。
