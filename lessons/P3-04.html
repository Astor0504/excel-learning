命名範圍與表格

本課進度 0 / 4
TL;DR — 30 秒看懂

命名範圍 + 表格：讓公式變得「會說人話」

=SUMIFS(C2:C500, A2:A500, "電子") 能算，但 =SUMIFS(業績表[金額], 業績表[部門], "電子") 半年後回來看也懂。命名範圍與表格的核心價值，就是讓公式可讀、可維護、會自動延伸。

命名範圍：替重要範圍取名字

選範圍 → 左上角「名稱方塊」輸入名字 → Enter。之後公式可以直接寫名字，例如把 B2:B100 取名為「業績」，公式變成 =SUM(業績)。適合匯率、稅率、參數區、固定查表範圍這類會被反覆引用的資料。

表格：讓資料區會自動長大

# 選資料 → ⌃+T 或 ⌃+L → 勾選「我的表格有標題」
# 表格會自動：
# 1. 加標題排序篩選按鈕
# 2. 新增列時公式自動套用
# 3. 範圍動態擴展，樞紐和圖表更穩定

# 結構化參照（公式裡用欄位名）
=SUM(業績表[金額])
=SUMIFS(業績表[金額], 業績表[部門], "電子")
=業績表[@金額] * 0.05  # @ 代表本列

普通範圍 vs 表格

維護差異比較
項目普通範圍表格
新增資料列公式可能沒有涵蓋自動納入表格範圍
公式可讀性C2:C500、A2:A500業績表[金額]、業績表[部門]
樞紐/圖表來源常要手動改範圍重新整理即可吃到新列
維護風險容易漏列或選錯欄欄名清楚，較不容易誤接

命名規則與維護邏輯

好的名稱要讓人知道資料的角色，而不是只知道位置。建議表格用名詞，例如 銷售表、商品表、客戶表；參數用明確用途，例如 稅率、匯率、目標毛利率。不要用 Table1、資料2 這類名稱，因為它們在公式裡沒有任何商業意義。

什麼時候用命名範圍，什麼時候用表格

兩者不是二選一，而是分工
情境建議理由
交易明細、訂單、客戶清單會新增列先轉成表格範圍會自動擴張，公式、樞紐、圖表都比較穩
稅率、匯率、目標值這類固定參數用命名範圍公式可讀性更好，調整參數也更集中
資料驗證來源清單表格或命名範圍都可清單會增減時用表格更省維護
跨工作表公式長到難讀表格 + 清楚名稱讓公式從位置語言變成資料語言

常見錯誤

表格名稱不要用空白或特殊符號，建議用 業績表、產品表、客戶表 這種清楚短名。欄位改名後，相關公式也會跟著更新，但如果刪掉整個欄位，公式仍可能失效；交付前要檢查錯誤值。

小技巧養成「先把原始資料轉成表格，再做公式、樞紐、圖表」的習慣。這不是儀式感，是降低後續維護成本。
