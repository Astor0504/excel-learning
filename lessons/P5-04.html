綜合挑戰：跨章節整合題目練習

把學過的全部用上。這些題目模擬真實工作場景——但每題開始前，請先回答一個問題：「這題我該用哪種工具？」能選對工具再動手，才是這課真正要驗收的能力。

工具選擇提示卡
  • 公式：單一欄位邏輯、條件判斷、查找對應值
  • 樞紐：快速彙總、切片、觀察分佈——先整理成 Table 再接
  • Power Query：重複清資料、多來源合併、可重跑的整形流程
  • VBA：Excel 內部操作自動化、報表交付流程、工作簿批次處理
  • Data Model / Power BI：多表關聯、量值治理、跨部門指標——Mac 上考慮跨 Windows 或 Power BI

挑戰 1：年資與獎金

先選工具：這是公式題。日期計算用 DATEDIF，條件判斷用 IFS，條件彙總用 SUMIFS。

員工表有「入職日期」和「年薪」兩欄：

  1. 計算每人滿幾年（DATEDIF 搭配 TODAY()）
  2. 用 COUNTIFS 算年資 ≥ 3 年的人數
  3. 用 IFS 按年資分級計算獎金比例，不要用巢狀 IF
  4. 用 SUMIFS 加總全公司應發獎金

驗收：你能不能在不用輔助欄的情況下，只靠公式直接輸出每人獎金？

挑戰 2：跨表查詢

先選工具：單表 JOIN → 公式（XLOOKUP）。彙總報表 → 樞紐。

訂單表只有「商品代號」，商品表有「代號 → 名稱 → 價格」：

  1. 用 XLOOKUP 帶出商品名稱與單價（注意：舊版環境改用 INDEX+MATCH）
  2. 計算每筆訂單小計（數量 × 單價）
  3. 兩張表都先轉成 Table，再接樞紐做月報
  4. 驗收：如果商品表新增一筆，訂單表的查找結果能不能自動更新？

挑戰 3：動態儀表板

先選工具：輸出清單用動態陣列（M365），排名統計用公式，視覺化用條件式格式 + 走勢圖。

=UNIQUE(銷售[業務員])                      # 列出所有業務員（動態，不用手動維護）
=SUMIFS(銷售[金額], 銷售[業務員], A2)      # 個人業績
=RANK(B2, B$2:B$20)                        # 排名（注意鎖定範圍）
=LARGE(銷售[金額], 1)                      # 最高單筆

驗收：新增一筆銷售資料後，儀表板的排名與加總有沒有自動更新？條件式格式有沒有跟著變？

挑戰 4：月報自動化系統

先選工具：合併多個 CSV → Power Query（可重跑）。彙總報表 → 樞紐。工作簿操作自動化 → VBA。

每月收到 N 個分公司的 .csv：

  1. Power Query：資料夾來源 → 合併所有 CSV → 整形清理 → Load 到工作表
  2. 樞紐：接 Query 的輸出做彙總，設定自動重新整理
  3. VBA：一鍵觸發 ActiveWorkbook.RefreshAll，再輸出指定工作表為 PDF
macOS 路線說明這題的主軸在 macOS 上完全可行：Power Query for Mac、樞紐、VBA 工作簿操作都支援。不要把重點放在 Windows 才有的 COM 郵件自動化（如用 VBA 控制 Outlook 寄信）——那不是這站教的路線，也不是 Mac 版的主路線。
