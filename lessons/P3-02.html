文字與日期函式

本課進度 0 / 4
TL;DR — 30 秒看懂

文字 + 日期清理：先去髒字元與空白，再拆欄位，最後把日期變成可計算的真日期

從系統匯出的資料常常不是「看起來髒」而已，而是混了不可見字元、奇怪空白、文字日期和格式不一致。這課的重點是建立一條穩定清理線，而不是零散背函數。

清資料的 4 步驟

先判斷資料問題，再選函數
步驟目的常用函數
清掉髒字元處理多餘空白、不可見字元、非斷行空白SUBSTITUTE / CLEAN / TRIM
拆出欄位從訂單號、Email、地址中抓出需要片段LEFT / MID / RIGHT / FIND
轉成真日期把 20260315 或 2026-03-15 變成 Excel 可計算日期DATE / DATEVALUE / VALUE
輸出顯示格式讓日期、金額、代碼以交付格式呈現TEXT / TEXTJOIN / &

文字清理起手式

# 最穩的空白清理：先處理非斷行空白，再清不可見字元，最後 TRIM
=TRIM(CLEAN(SUBSTITUTE(A2, CHAR(160), "")))

# Email 取帳號與網域
=LEFT(A2, FIND("@", A2)-1)
=MID(A2, FIND("@", A2)+1, 100)

# 訂單號 A2026031101：取年份、月份、日期
=MID(A2, 2, 4)
=MID(A2, 6, 2)
=MID(A2, 8, 2)

# 電話去掉符號
=SUBSTITUTE(A2, "-", "")

日期處理起手式

# 訂單號 A2026031101 → 真正日期
=DATE(MID(A2,2,4), MID(A2,6,2), MID(A2,8,2))

# 文字日期 → 真正日期
=DATEVALUE(A2)

# 年資、工作天、月底
=DATEDIF(A2, TODAY(), "Y")
=WORKDAY(A2, 5)
=EOMONTH(A2, 0)

# 只改顯示格式，結果會變文字
=TEXT(A2, "yyyy-mm-dd")

TEXT 的陷阱

TEXT 很適合輸出報表顯示格式，例如把 2026/3/1 顯示成 2026-03。但 TEXT 的結果是文字，不是日期本身。若後面還要排序、加減天數或做日期軸，請保留原本的真日期欄，再另外做顯示欄。

實戰範例：從訂單號建立可計算日期

# A2 = A2026031101
=DATE(
  MID(A2, 2, 4),
  MID(A2, 6, 2),
  MID(A2, 8, 2)
)
# 結果是 2026/03/11，可拿來排序、分月、算天數
⚠️ 注意DATEDIF 是隱藏函數，輸入時不會自動提示，但能用。MID 從 1 開始算，不是 0；如果你從程式語言背景來，這裡很容易手滑。
