Power Pivot 與資料模型

本課進度 0 / 4
TL;DR — 30 秒看懂

Power Pivot + DAX：把 Excel 變成迷你 BI 工具

資料超過樞紐處理上限（約百萬列）？要跨多張表分析？這時 Power Pivot 就出場了。

資料模型

傳統樞紐只能用「一張表」。資料模型可以匯入多張表 → 設定關聯（像資料庫的 JOIN）→ 建立樞紐時跨表分析。

DAX 基礎

# 度量值（Measures）= 公式語言版的計算欄位
總業績 := SUM(訂單[金額])

電子部業績 :=
    CALCULATE(
        SUM(訂單[金額]),
        部門[部門名] = "電子"
    )

# 同期比較
去年同期 :=
    CALCULATE([總業績], SAMEPERIODLASTYEAR('日期'[日期]))

和 Power Query 的差別

Power Query：清資料、整形資料；Power Pivot：建模型、寫 DAX、做分析。一個負責「清」，一個負責「算」，常常一起用。

⚠️ 注意Power Pivot 的資料模型 macOS 版支援有限，部分功能要 Windows 版。Power Query 在 Mac 已經完整支援。
