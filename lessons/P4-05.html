Power Pivot 與資料模型

本課進度 0 / 4
TL;DR — 30 秒看懂

Power Pivot + DAX：把 Excel 變成迷你 BI 工具

資料超過樞紐處理上限、需要跨多張表關聯分析、或要建立可重複使用的量值計算——這時就進入資料模型的思維範疇。對 macOS 使用者來說，這課的價值是建立觀念與判斷力，而不是操作 Power Pivot 的路線指引。

為什麼要學這個概念

傳統樞紐只能用「一張表」。當你開始遇到以下情境，就代表已經走到資料模型的邊界：

  • 訂單表、商品表、客戶表分開存放，需要跨表彙總分析
  • 資料列數超過數十萬，樞紐效能開始下降
  • 需要定義「量值」（如毛利率、同期比較）供多個樞紐共用
  • 團隊需要一致的指標定義，而不是每人用不同公式各自計算

這時候的解法路線是：Power Query 負責清資料 → Data Model 負責建關聯與量值 → 樞紐只負責展示。

核心概念：資料模型與 DAX

資料模型（Data Model）允許你把多張表匯入，設定表與表之間的關聯（類似資料庫的 JOIN），再建立跨表樞紐分析。DAX（Data Analysis Expressions）是在模型上寫量值的公式語言：

# 量值（Measures）= 定義好的計算邏輯，可被多個樞紐共用
總業績 := SUM(訂單[金額])

電子部業績 :=
    CALCULATE(
        SUM(訂單[金額]),
        部門[部門名] = "電子"
    )

# 同期比較（Time Intelligence）
去年同期 :=
    CALCULATE([總業績], SAMEPERIODLASTYEAR('日期'[日期]))

Power Query vs Data Model：分工定位

工具負責什麼何時用
Power Query清資料、整形、合併來源資料進來時先過一次
Data Model / Power Pivot建關聯、定義量值、多表分析需要跨表、大量資料、共用指標時
樞紐分析切片、彙總、展示最後一步，接資料模型的輸出
⚠️ macOS 主路線說明依 Microsoft 官方說明，Power Pivot 不包含在 Office for Mac。這意味著在 Mac 上你無法直接操作 Power Pivot 介面或建立 DAX 量值。這課對 Mac 使用者的實務價值，是讓你理解資料模型的思維，知道：
  • 什麼情況該請 Windows 同事協作或開 VM
  • 什麼情況可以先用 Power Query + 樞紐的組合撐住
  • 什麼情況應該直接跨到 Power BI（完全支援 Mac 瀏覽器使用）

Mac 使用者的實務決策路線

  1. 先問規模：資料是幾萬列以內、單一來源？→ Power Query + 樞紐就夠，不需要資料模型。
  2. 再問關聯：需要跨兩張以上的表、有 ID 可以 JOIN？→ 先評估是否用 XLOOKUP / SUMIFS 拼起來，夠用就不往資料模型走。
  3. 再問量值：需要毛利率、同期比較等跨多個樞紐共用的指標？→ 這才是真正需要 Data Model 的訊號，考慮 Windows 環境或 Power BI。
  4. 最後評估工具：Power BI Desktop 支援完整的 Data Model + DAX，且 Mac 可透過瀏覽器查看報表；若團隊重度依賴這類分析，這才是比學 Mac 版 Excel 更值得投入的方向。
